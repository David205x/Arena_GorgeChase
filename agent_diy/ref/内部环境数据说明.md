# 峡谷追猎（Gorge Chase）环境数据说明

> 面向强化学习算法开发，聚焦算法工程师实际使用的数据。
>
> 本文档基于 `kaiwu_arena` 实际处理链路（`StandardEnv → EnvAdapter → KaiwuDataHandler`）整理，
> 描述算法 Agent 最终能够拿到的数据结构。

---

## 一、场景概述

**目标**：控制英雄小悟在有限步数内尽可能多地收集宝箱并存活，同时躲避追逐的怪物。

**得分规则**：`total_score = step_score + treasure_score`，其中 `step_score = step_no × step_score_multiplier`（默认 1.5），`treasure_score = treasures_collected × treasure_value`（默认每个宝箱 100 分）。环境原始奖励为每步 total_score 增量：`reward = total_score(t) - total_score(t-1)`。

**终止条件**：

| 条件 | terminated | truncated | 说明 |
|------|:---------:|:---------:|------|
| 达到最大步数（`step_no >= max_step`） | `False` | `True` | 正常完成对局 |
| 怪物与英雄同格（被捕获） | `True` | `False` | 失败终止，局部得分仍保留 |
| 接口错误 / 异常截断 | `False` | `True` | `result_code != 0` |

---

## 二、数据处理链路

```
底层环境 (Lua Gamecore)
    └─ libenv_client_manager
        └─ StandardEnv._parse_env_obs()          # 字段清洗
        │    - 删除 env_info 中: finished_steps, map_random, total_map
        │    - 重命名: buff_obtained_count → collected_buff
        └─ EnvAdapter._build_env_response()       # 奖励计算
        │    - reward = total_score(t) - total_score(t-1)
        └─ KaiwuDataHandler.observation_process() # 字段重命名 + 视野裁剪
             - 重命名: legal_act → legal_action
             └─ get_local_frame_state()           # 视野裁剪（切比雪夫距离 ≤ 10）
                  - monsters: 视野外保留条目，pos={-1,-1}, speed=-1，新增 is_in_view
                  - organs: 视野外直接移除
                  - 移除 monsters[].start_pos
                  └─ 算法 Agent 最终数据
```

**关键处理说明：**
- `StandardEnv` 从 `env_info` 中删除了 `finished_steps`、`map_random`、`total_map`
- `StandardEnv` 将 `buff_obtained_count` 重命名为 `collected_buff`
- `DataHandler` 将 `legal_act` 重命名为 `legal_action`
- `DataHandler` 新增 `monsters[].is_in_view` 视野标记字段（底层数据中不存在）
- **视野过滤规则**：切比雪夫距离（正方形区域），半径 10，即 21×21 范围

## 三、最终数据结构（算法可访问）

### 3.1 接口调用方式

```python
# reset 接口 — 返回初始观测（直接是 obs dict，不是 {"obs": ...} 包一层）
obs = env.reset(usr_conf=usr_conf)

# step 接口 — 返回 (env_info, done)
env_info, done = env.step(action)   # action: int, [0, 15]

# 提取奖励和观测
r   = env_info["reward"]["reward"]  # float，本步增量得分
obs = env_info["obs"]               # dict，观测数据
```

> **⚠️ 注意**：`reset` 直接返回 `obs dict`，而 `step` 返回 `(env_info, done)`，两者结构不同。

### 3.2 完整结构总览

```python
# step() 返回的 env_info 标准结构
env_info = {
    "reward": {
        "env_id":   str,    # [平台约定] 环境标识
        "frame_no": int,    # [平台约定] 当前帧号
        "reward":   float,  # 本步增量奖励 = total_score(t) - total_score(t-1)
    },
    "obs": {
        "env_id":      str,   # [平台约定] 环境标识
        "frame_no":    int,   # [平台约定] 当前帧号
        "observation": {      # result_code==0 时有效，否则为 None
            "step_no":      int,              # 当前步数（从 1 开始递增）
            "frame_state":  dict,             # 局部视野帧状态（经切比雪夫视野过滤）
            "env_info":     dict,             # 全局统计信息
            "map_info":     list[list[int]],  # 21×21 局部视野地图
            "legal_action": list,             # 合法动作掩码（由 legal_act 重命名）
        },
        "extra_info": {       # 补充信息（调试/评估用）
            "frame_state":    dict,  # 完整帧状态（未经视野过滤）
            "map_id":         int,   # 当前对局地图编号
            "result_code":    int,   # [平台约定] 0=正常，非0=错误
            "result_message": str,   # [平台约定] 错误描述
        },
        "terminated":  bool,  # [平台约定] 被怪物捕获时为 True
        "truncated":   bool,  # [平台约定] 达到最大步数或异常时为 True
    }
}

done = terminated or truncated
```

| `truncated` | `obs.truncated` | `bool` | 是 | `true`/`false` | 达到最大步数或异常时为 True | 底层环境 |
| `step_no` | `obs.observation.step_no` | `int` | 是 | `[1, max_step]` | 当前步数（从 1 开始） | 底层环境 |
| `result_code` | `obs.extra_info.result_code` | `int` | 是 | 见错误码表 | 0=正常，非0=错误 | EnvAdapter |
| `result_message` | `obs.extra_info.result_message` | `str` | 是 | — | 错误描述信息 | EnvAdapter |

### 3.4 observation（主要特征数据）

#### 3.4.1 frame_state — 实体状态

> **⚠️ 每步必查**：先判断 `obs["extra_info"]["result_code"] == 0` 再访问 `observation`，否则 `observation` 为 `None` 会报错。

##### heroes（英雄小悟，dict 类型）

> **⚠️ heroes 是 dict，不是 list**。请用 `frame_state["heroes"]["pos"]`，而不是 `frame_state["heroes"][0]["pos"]`。

```python
frame_state["heroes"] = {
    "hero_id":                  int,    # 英雄实体 ID
    "pos": {
        "x": int,                       # x 坐标（横向）[0, 127]
        "z": int,                       # z 坐标（纵向）[0, 127]
    },
    "flash_cooldown":           int,    # 闪现技能剩余冷却步数（0=可立即使用）
    "buff_remaining_time":      int,    # 加速 buff 剩余步数（0=无 buff）
    # 以下字段与 env_info 中对应字段值一致（冗余字段）
    "treasure_score":           float,  # 宝箱得分（同 env_info.treasure_score）
    "step_score":               float,  # 步数得分（同 env_info.step_score）
    "treasure_collected_count": int,    # 已收集宝箱数（同 env_info.treasures_collected）
}
```

| 字段名 | 字段路径 | 类型 | 是否必填 | 取值范围 | 字段说明 | 数据来源 |
|--------|---------|------|:--------:|---------|---------|---------|
| `hero_id` | `obs.observation.frame_state.heroes.hero_id` | `int` | 是 | — | 英雄实体 ID | 环境侧 |
| `pos.x` | `obs.observation.frame_state.heroes.pos.x` | `int` | 是 | `[0, 127]` | x 坐标（横向，向右增大） | 环境侧 |
| `pos.z` | `obs.observation.frame_state.heroes.pos.z` | `int` | 是 | `[0, 127]` | z 坐标（纵向，向下增大） | 环境侧 |
| `flash_cooldown` | `obs.observation.frame_state.heroes.flash_cooldown` | `int` | 是 | `[0, talent_cooldown]` | 闪现技能剩余冷却步数，0=可立即使用 | 环境侧 |
| `buff_remaining_time` | `obs.observation.frame_state.heroes.buff_remaining_time` | `int` | 是 | `[0, buff_duration]` | 加速 buff 剩余步数，0=无 buff | 环境侧 |
| `treasure_score` | `obs.observation.frame_state.heroes.treasure_score` | `float` | 是 | `≥0` | 宝箱得分（同 `env_info.treasure_score`） | 环境侧 |
| `step_score` | `obs.observation.frame_state.heroes.step_score` | `float` | 是 | `≥0` | 步数得分（同 `env_info.step_score`） | 环境侧 |
| `treasure_collected_count` | `obs.observation.frame_state.heroes.treasure_collected_count` | `int` | 是 | `≥0` | 已收集宝箱数（同 `env_info.treasures_collected`） | 环境侧 |

##### monsters（怪物列表，经视野过滤）

`monsters` 列表**始终保留全部怪物条目**，视野外怪物以 `is_in_view=0` 标记并屏蔽位置/速度信息。

**列表长度规则**：
- 对局开始至 `monster_interval` 步前：长度为 1（仅第一只怪物）
- `monster_interval` 步后：长度为 2（第二只怪物生成后加入）
- `monster_count=1` 或 `monster_interval=-2` 时：全程长度为 1

```python
frame_state["monsters"] = [
    {
        "monster_id":           int,  # 怪物实体 ID（第一只≈1，第二只≈2）
        "is_in_view":           int,  # 1=在视野内，0=视野外 [DataHandler 新增字段]
        "pos": {
            "x": int,                 # 视野内：真实 x 坐标；视野外：固定为 -1
            "z": int,                 # 视野内：真实 z 坐标；视野外：固定为 -1
        },
        "speed":                int,  # 视野内：当前速度（格/步）；视野外：固定为 -1
        "monster_interval":     int,  # 第二怪物生成间隔配置（只读，不随步骤变化）
        # ⚠️ start_pos 字段已被 local_view 移除，不可从此处访问，如需请从 extra_info.frame_state 获取
    },
    ...  # 最多 2 只怪物
]
```

**视野过滤规则**：
- 过滤算法：切比雪夫距离（正方形区域）
- 视野半径：10（来自 `game.yaml` 中的 `view` 配置）
- 视野内（`is_in_view=1`）：`pos`、`speed` 为真实值
- 视野外（`is_in_view=0`）：`pos` 固定为 `{"x": -1, "z": -1}`，`speed` 固定为 `-1`

| 字段名 | 字段路径 | 类型 | 是否必填 | 取值范围/枚举 | 字段说明 | 数据来源 |
|--------|---------|------|:--------:|-------------|---------|---------|
| `monster_id` | `obs.observation.frame_state.monsters[i].monster_id` | `int` | 是 | — | 怪物实体 ID | 环境侧 |
| `is_in_view` | `obs.observation.frame_state.monsters[i].is_in_view` | `int` | 是 | `0` / `1` | 1=在视野内，0=视野外 | DataHandler 新增 |
| `pos.x` | `obs.observation.frame_state.monsters[i].pos.x` | `int` | 是 | `[0,127]` 或 `-1` | 视野内：真实 x 坐标；视野外：-1 | 环境侧 / DataHandler 屏蔽 |
| `pos.z` | `obs.observation.frame_state.monsters[i].pos.z` | `int` | 是 | `[0,127]` 或 `-1` | 视野内：真实 z 坐标；视野外：-1 | 环境侧 / DataHandler 屏蔽 |
| `speed` | `obs.observation.frame_state.monsters[i].speed` | `int` | 是 | `[0,4]` 或 `-1` | 视野内：当前速度（格/步）；视野外：-1 | 环境侧 / DataHandler 屏蔽 |
| `monster_interval` | `obs.observation.frame_state.monsters[i].monster_interval` | `int` | 是 | `-2, -1, [11,2000]` | 第二怪物生成间隔配置（只读） | 环境侧 |

> **算法使用前必须先判断** `monsters[i]["is_in_view"] == 1`，视野外时 `pos` 和 `speed` 均为 -1，不代表真实状态。

##### organs（物件列表：宝箱/加速 buff，仅保留视野内）

organs **只保留视野内的物件**，视野外的物件直接从列表移除（与 monsters 处理方式不同）。

**列表长度规则**：长度动态变化，取决于视野内物件数量；宝箱被收集后永久从列表消失；buff 被收集后 `status=0`，刷新后恢复为 `status=1`。

```python
frame_state["organs"] = [
    {
        "sub_type":                 int,    # 物件类型：1=宝箱（Chest），2=加速 buff（SpeedUpBuff）
        "config_id":                int,    # 物件固定 ID（用于跨步骤追踪同一物件）
        "status":                   int,    # 1=存在/可收集，0=已被收集（buff 进入刷新倒计时）
        "pos": {
            "x": int,                       # 绝对格子坐标 x
            "z": int,                       # 绝对格子坐标 z
        },
        "hero_l2_distance":         int,    # 与英雄的 L2 距离桶编号（0-5）
        "hero_relative_direction":  int,    # 相对于英雄的方向（0-8）
    },
    ...
]
```

| 字段名 | 字段路径 | 类型 | 是否必填 | 取值范围/枚举 | 字段说明 | 数据来源 |
|--------|---------|------|:--------:|-------------|---------|---------|
| `sub_type` | `obs.observation.frame_state.organs[i].sub_type` | `int` | 是 | `1`=宝箱，`2`=加速 buff | 物件类型 | 环境侧 |
| `config_id` | `obs.observation.frame_state.organs[i].config_id` | `int` | 是 | — | 物件固定 ID，可跨步骤追踪同一物件 | 环境侧 |
| `status` | `obs.observation.frame_state.organs[i].status` | `int` | 是 | `0`=已收集/刷新中，`1`=存在可收集 | 物件当前状态 | 环境侧 |
| `pos.x` | `obs.observation.frame_state.organs[i].pos.x` | `int` | 是 | `[0, 127]` | 绝对 x 坐标 | 环境侧 |
| `pos.z` | `obs.observation.frame_state.organs[i].pos.z` | `int` | 是 | `[0, 127]` | 绝对 z 坐标 | 环境侧 |
| `hero_l2_distance` | `obs.observation.frame_state.organs[i].hero_l2_distance` | `int` | 是 | `[0, 5]` | 与英雄的 L2 距离桶编号（每桶约 30 格） | 环境侧 |
| `hero_relative_direction` | `obs.observation.frame_state.organs[i].hero_relative_direction` | `int` | 是 | `0`=同格/不可达，`1`=东，`2`=东北，`3`=北，`4`=西北，`5`=西，`6`=西南，`7`=南，`8`=东南 | 相对于英雄的方向 | 环境侧 |

> **注意**：`extra_info.frame_state["organs"]` 包含**所有**物件（包含视野外的），供需要全局信息的算法使用。

#### 3.4.2 env_info — 全局统计

```python
observation["env_info"] = {
    # === 得分 ===
    "total_score":    float,  # 当前总得分 = step_score + treasure_score
    "step_score":     float,  # 步数得分 = step_no × step_score_multiplier（默认 1.5）
    "treasure_score": float,  # 宝箱得分 = treasures_collected × treasure_value（默认 100）

    # === 英雄位置（与 frame_state.heroes.pos 值相同）===
    "pos": {
        "x": int,   # 英雄当前 x 坐标（同 frame_state.heroes.pos.x）
        "z": int,   # 英雄当前 z 坐标（同 frame_state.heroes.pos.z）
    },

    # === 宝箱统计 ===
    "treasures_collected": int,       # 已收集宝箱数（同 frame_state.heroes.treasure_collected_count）
    "total_treasure":      int,       # 本局宝箱总数（由 treasure_count 配置，运行时不变）
    "treasure_id":         list[int], # 当前场上剩余宝箱的 config_id 列表

    # === 闪现（技能）===
    "flash_count":        int,  # 本局闪现使用总次数
    "flash_cooldown_max": int,  # 闪现冷却上限（由 talent_cooldown 配置，运行时不变）

    # === 加速 buff ===
    "collected_buff":    int,  # 本局拾取 buff 总次数（原字段名 buff_obtained_count，StandardEnv 已重命名）
    "total_buff":        int,  # 本局 buff 总数（由 buff_count 配置，运行时不变）
    "buff_refresh_time": int,  # buff 刷新时间（由 buff_cooldown 配置，运行时不变）

    # === 怪物配置（只读，运行时不变）===
    "monster_interval":         int,  # 第二怪物生成间隔配置
    "monster_init_speed":       int,  # 怪物初始速度配置
    "monster_speed_boost_step": int,  # 怪物加速增益步数配置

    # === 对局进度 ===
    "max_step": int,  # 最大步数配置（运行时不变）
    "step_no":  int,  # 当前步数（与 observation.step_no 相同，冗余字段）
}

# ⚠️ 以下字段已被 StandardEnv 删除，不可访问：
# "finished_steps"  → 已被删除，使用 step_no 代替
# "map_random"      → 已被删除，不暴露
# "total_map"       → 已被删除，不暴露
```

| `monster_speed_boost_step` | `obs.observation.env_info.monster_speed_boost_step` | `int` | 是 | `-1,[1,2000]` | 怪物加速增益步数配置（只读） | 环境侧 |
| `max_step` | `obs.observation.env_info.max_step` | `int` | 是 | `[1, 2000]` | 最大步数配置（不随步骤变化） | 环境侧 |
| `step_no` | `obs.observation.env_info.step_no` | `int` | 是 | `[1, max_step]` | 当前步数（同 `observation.step_no`，冗余字段） | 环境侧 |

#### 3.4.3 map_info — 局部视野地图

```python
observation["map_info"] = list[list[int]]
# 尺寸：21×21（view_size=10，直径 = 2×10+1）
# 索引：map_info[row][col]，row 对应 z 方向，col 对应 x 方向
# 中心：map_info[10][10] 对应英雄当前位置
```

| 值 | 含义 |
|:--:|------|
| `0` | 不可通行（障碍物/地图边界） |
| `1` | 可通行区域 |

**坐标映射关系**：
- `map_info[10][10]`：英雄当前位置（中心点）
- `map_info[10][11]`：英雄右侧 1 格（x+1）
- `map_info[9][10]`：英雄上方 1 格（z-1）
- 超出地图边界的格子固定为 `0`（不可通行）

```python
import numpy as np

# numpy 转换示例
map_array = np.array(observation["map_info"])  # shape: (21, 21)

# 判断某方向是否可通行（dx, dz 为方向向量）
def is_passable(map_info, dx, dz):
    center = len(map_info) // 2  # = 10
    row, col = center + dz, center + dx
    if 0 <= row < len(map_info) and 0 <= col < len(map_info[0]):
        return map_info[row][col] != 0
    return False
```

#### 3.4.4 legal_action — 合法动作掩码

```python
observation["legal_action"] = list  # 由底层 legal_act 字段重命名而来（DataHandler 处理）
# 长度：16（对应 16 个动作）
```

**格式说明**（两种可能，取决于底层版本）：
- **布尔掩码**（`list[bool]`，长度 16）：`[True, True, ..., False, False]`，索引即动作 ID
- **索引列表**（`list[int]`）：`[0, 1, 2, ..., 7]` 表示合法动作的索引

**语义规则**：
- 动作 0-7（移动）：**恒为合法**（True）
- 动作 8-15（闪现）：仅当 `frame_state.heroes.flash_cooldown == 0` 时为合法

**通用处理方法**：

```python
def get_legal_indices(legal_action):
    if not legal_action:
        return list(range(8))  # 默认至少移动动作合法
    if isinstance(legal_action[0], bool):
        return [i for i, v in enumerate(legal_action) if v]
    return [int(a) for a in legal_action]
```

> **注意**：底层字段名为 `legal_act`，DataHandler 处理时重命名为 `legal_action`。

obs["extra_info"] = {

**移动规则**：
1. **基础速度**：每步移动 1 格（`player_speed_normal=1`）；持有加速 buff 时速度额外 +1（即可移动 2 格）

5. **闪现距离**：直线最远 10 格（`flash_distance=10`），斜向最远 8 格（`flash_distance_diagonal=8`）；遇墙在最远可通行位置停止

## 六、实体位置关系与距离计算

```python
import math

# 英雄位置
hero_pos = obs_data["env_info"]["pos"]     # {"x": int, "z": int}
hx, hz   = hero_pos["x"], hero_pos["z"]

# 切比雪夫距离（视野判断用）
def chebyshev_dist(pos1, pos2):
    return max(abs(pos1["x"] - pos2["x"]), abs(pos1["z"] - pos2["z"]))

# 欧氏距离（L2 距离，通用距离计算）
def euclidean_dist(pos1, pos2):
    return math.sqrt((pos1["x"] - pos2["x"])**2 + (pos1["z"] - pos2["z"])**2)

# L2 距离桶编号（0-5，每桶约 30 格，对应 128×128 地图最大对角线约 180 格）
# organs[i]["hero_l2_distance"] 和 monsters[i] 中使用的距离桶编号

# 到最近宝箱的距离
organs = obs_data["frame_state"]["organs"]
chests = [o for o in organs if o["sub_type"] == 1 and o["status"] == 1]
if chests:
    closest_chest = min(chests, key=lambda o: euclidean_dist({"x": hx, "z": hz}, o["pos"]))
    dist_to_chest = euclidean_dist({"x": hx, "z": hz}, closest_chest["pos"])

# 到视野内最近怪物的距离
monsters_in_view = [m for m in obs_data["frame_state"]["monsters"] if m["is_in_view"] == 1]
if monsters_in_view:
    closest_monster = min(monsters_in_view, key=lambda m: euclidean_dist({"x": hx, "z": hz}, m["pos"]))
    dist_to_monster = euclidean_dist({"x": hx, "z": hz}, closest_monster["pos"])
```

## 八、环境逻辑与关键规则

### 8.1 得分规则

| 得分类型 | 计算公式 | 说明 |
|---------|---------|------|
| 步数得分 | `step_no × step_score_multiplier`（默认 1.5） | 每步线性增长 |
| 宝箱得分 | `treasures_collected × treasure_value`（默认 100） | 收集宝箱时阶跃增加 |
| 总得分 | `step_score + treasure_score` | 两部分之和 |

### 8.2 收集规则

| 物件类型 | sub_type | 收集触发 | 收集效果 | 收集后状态 |
|---------|:--------:|---------|---------|----------|
| 宝箱（Chest） | `1` | 经过宝箱所在格子 | `+treasure_value` 分（默认 +100） | `status=0`，永久消失 |
| 加速 buff（SpeedUpBuff） | `2` | 经过 buff 所在格子 | 速度额外 +`buff_extra_speed`（默认 +1），持续 `buff_duration`（默认 50 步） | `status=0`，`buff_refresh_time`（默认 200 步）后恢复 `status=1` |

### 8.3 怪物规则

| 规则 | 说明 |
|------|------|
| 第一只怪物 | 对局开始时生成，出生点为地图指定位置或右下区域 |
| 第二只怪物 | 在 `monster_interval` 步（默认 300）时于玩家 10 步前位置生成；`monster_interval=-2` 时不生成 |
| 移动方式 | 使用 A* 寻路算法追逐玩家，每步移动 `monster_speed` 格 |
| 加速增益 | 到达 `monster_speed_boost_step`（默认 500 步）后，所有怪物速度 +1 |
| 碰撞判定 | 怪物与玩家同格即判定失败（`terminated=True`） |
| 防重叠 | 怪物之间不会重叠（同一格子最多一只怪物） |

### 8.4 终止规则

```python
# 对局终止的两种情况
terminated = True   # 被怪物捕获（失败）
truncated  = True   # 达到最大步数（正常完成）或接口异常

done = terminated or truncated
```

## 九、访问示例

```python
def parse_env_obs(env_info: dict) -> dict:
    """
    从 step() 返回的 env_info 中提取所有关键数据。

    Returns:
        dict: 包含所有提取的关键数据，未提取成功时对应字段为 None
    """
    obs = env_info["obs"]

    # 1. 判断终止信号
    terminated = obs["terminated"]
    truncated  = obs["truncated"]
    done       = terminated or truncated

    # 2. 提取奖励
    reward = env_info["reward"]["reward"]

    # 3. 先检查 result_code，再访问 observation
    result_code    = obs["extra_info"]["result_code"]
    result_message = obs["extra_info"]["result_message"]
    if result_code != 0:
        # ⚠️ obs["observation"] 此时为 None，不能访问
        return {
            "done": done,
            "reward": reward,
            "result_code": result_code,
            "result_message": result_message,
            "observation": None,
        }

    observation = obs["observation"]
    frame_state = observation["frame_state"]
    env_info_data = observation["env_info"]
    heroes = frame_state["heroes"]

    # 4. 提取英雄状态
    hero_pos = heroes["pos"]                         # {"x": int, "z": int}
    flash_cooldown = heroes["flash_cooldown"]        # 0=可立即使用
    buff_remaining = heroes["buff_remaining_time"]   # 0=无 buff

    # 5. 提取怪物状态（注意视野过滤）
    monsters_data = []
    for monster in frame_state["monsters"]:
        monsters_data.append({
            "monster_id": monster["monster_id"],
            "is_in_view": monster["is_in_view"],
            "pos": monster["pos"] if monster["is_in_view"] == 1 else None,
            "speed": monster["speed"] if monster["is_in_view"] == 1 else None,
        })

    # 6. 提取视野内物件（organs 已经只包含视野内的）
    chests = [o for o in frame_state["organs"] if o["sub_type"] == 1 and o["status"] == 1]
    buffs  = [o for o in frame_state["organs"] if o["sub_type"] == 2 and o["status"] == 1]

    # 7. 提取全局统计
    total_score        = env_info_data["total_score"]
    step_no            = observation["step_no"]
    treasures_collected = env_info_data["treasures_collected"]
    total_treasure     = env_info_data["total_treasure"]

    # 8. 提取合法动作（兼容两种格式）
    legal_action = observation["legal_action"]
    if legal_action and isinstance(legal_action[0], bool):
        legal_indices = [i for i, v in enumerate(legal_action) if v]
    else:
        legal_indices = [int(a) for a in legal_action] if legal_action else list(range(8))

    # 9. 提取局部地图
    map_info = observation["map_info"]  # list[list[int]]，21×21

    # 10. 提取 extra_info（调试/评估用）
    map_id = obs["extra_info"]["map_id"]

    return {
        "done": done,
        "terminated": terminated,
        "truncated": truncated,
        "reward": reward,
        "result_code": result_code,
        "step_no": step_no,
        "hero_pos": hero_pos,
        "flash_cooldown": flash_cooldown,
        "buff_remaining": buff_remaining,
        "monsters": monsters_data,
        "chests_in_view": chests,
        "buffs_in_view": buffs,
        "total_score": total_score,
        "treasures_collected": treasures_collected,
        "total_treasure": total_treasure,
        "legal_indices": legal_indices,
        "map_info": map_info,
        "map_id": map_id,
    }
```

## 十一、常见陷阱

1. **heroes 是 dict 不是 list**：`frame_state["heroes"]` 是单个 dict，不要用 `frame_state["heroes"][0]`。
2. **observation 可能为 None**：`result_code != 0` 时 `observation` 为 `None`，必须先检查 `result_code` 再访问。
3. **怪物视野外坐标为 (-1, -1)**：`is_in_view=0` 的怪物 `pos.x` 和 `pos.z` 均为 -1，不代表真实位置，使用前必须判断 `is_in_view`。
4. **怪物视野外速度为 -1**：`is_in_view=0` 的怪物 `speed=-1`，不代表真实速度。
5. **organs 只包含视野内的物件**：视野外的宝箱/buff 不出现在 `observation.frame_state.organs` 中（与 monsters 不同），如需全局物件信息请用 `extra_info.frame_state.organs`。
6. **start_pos 字段已被移除**：`observation.frame_state.monsters` 中不含 `start_pos`，需要时请从 `extra_info.frame_state.monsters` 获取。
7. **字段重命名 legal_act → legal_action**：底层字段名为 `legal_act`，DataHandler 处理后重命名为 `legal_action`，不可在 `observation` 中访问 `legal_act`。
8. **字段重命名 buff_obtained_count → collected_buff**：底层字段名为 `buff_obtained_count`，StandardEnv 处理后重命名为 `collected_buff`。
9. **配置字段名与 env.md 不一致**：`train_env_conf.toml` 中 `talent_cooldown`（非 `flash_cooldown`）、`buff_cooldown`（非 `buff_refresh_time`）、`monster_speedup`（非 `monster_speed_boost_step`）。
10. **已删除字段不可访问**：`finished_steps`、`map_random`、`total_map` 已被 StandardEnv 从 `env_info` 中删除。
11. **第二只怪物动态生成**：`monster_interval` 步前 `monsters` 列表长度为 1，不要假设始终有 2 只怪物。
12. **奖励是增量而非绝对值**：`reward["reward"]` 是 `total_score(t) - total_score(t-1)`，每步约 +1.5（步数分），收集宝箱时额外 +100；非稀疏奖励，但宝箱分比例大。
13. **step_no 从 1 开始**：`observation["step_no"]` 从 1 开始递增，不是从 0 开始。
14. **is_in_view 字段是 DataHandler 新增的**：底层原始数据中不存在 `is_in_view` 字段，由 `local_view.py` 的 `get_local_frame_state()` 新增。

## 附录：错误码说明

| 错误码 | 错误类型 | 说明 | 处理建议 |
|:------:|---------|------|---------|
| `0` | 正常 | 接口正常返回 | — |
| `-100` | reset 配置错误 | reset 配置创建失败 | 检查 `usr_conf` 格式和字段完整性 |
| `-101` | 配置验证失败 | 配置参数非法（类型/范围错误） | 检查 `env_conf` 中各参数取值范围 |
| `-103` | reset 执行异常 | reset 过程中发生未预期错误 | 检查日志，联系环境开发者 |
| `-1` | step 执行异常 | step 过程中发生未预期错误 | 调用 reset 重置环境 |
| `-201` | 环境状态异常 | 环境已结束但未 reset 就调用 step | 先调用 reset 重置 |

```python
# 错误处理示例
result_code = env_info["obs"]["extra_info"]["result_code"]
if result_code != 0:
    # ⚠️ obs["observation"] 此时为 None，不能访问
    error_msg = env_info["obs"]["extra_info"]["result_message"]
    print(f"环境接口错误 [{result_code}]: {error_msg}")
    # 调用 reset 重置环境
    obs = env.reset(usr_conf=usr_conf)
```
