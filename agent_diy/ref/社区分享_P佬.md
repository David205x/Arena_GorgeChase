- P佬认为 **怪物加速前** 和 **怪物加速后** 两个阶段决策模式存在差异，故指标也分开统计
  - 第二阶段生存压力显著增大，容易暴毙

  - 否则无法验证是过拟合现有地图还是模型的泛化性

## 监控指标

> 这一部分是原文第一部分讲的，配上监控图确实很有说服力。这一部分应该是搞清楚 “如何设计指标以清晰体现agent学习状态”，以及 “我真正关注的是哪些指标”

### 算法

> 这一部分比较常规，PPO一般都会关注这些

- `CumReward`：当前训练 batch 的平均奖励。
- `TotalLoss`：PPO 总损失。
- `PolicyLoss`：策略网络损失，用来看 actor 的更新情况。
- `ValueLoss`：价值网络损失，用来看 critic 学得稳不稳。
- `EntropyLoss`：策略熵，反映当前动作分布的随机性，太低通常说明探索不够。
- `GradClipNorm`：梯度裁剪前的梯度范数，可以辅助判断训练是否过激。
- `ClipFrac`：PPO 中被 clip 的比例，过高通常说明策略更新过猛。
- `ExplainedVar`：value 对回报的解释程度，越高通常说明 critic 拟合越好。

> 👆 这个我好像平常没有关注
>
> 💡 衡量 critic 预测的 value 与真实回报（returns/GAE target）之间的拟合程度
>
> 💡 ExplainedVar = 1 - var(G - value) / var(G)

- `AdvMean`：当前 batch 的 advantage 均值。
- `RetMean`：当前 batch 的 return 均值。

## 特征工程

> 被P佬评价为 **最有效的部分**，即性价比收益最高的部分

智能体在决策时，到底真正需要看到什么？

### P佬的先验切入点

这个题的 observation 不能只回答“环境里有什么”，还要尽量回答：

- 我现在危险不危险
- 我现在有没有机会拿资源
- 我往哪里走更安全
- 我是应该继续拿分，还是该准备保命

P佬的感觉是：如果特征能同时把“危险”和“机会”说清楚，模型会聪明很多。

### 提到的观测构成

- 英雄主特征：主要负责回答“我现在是什么状态”。 
  - 环境信息：当前位置相关状态、当前分数进展、闪现是否可用等
  - 先验指标：已经进入第几个阶段、最近是否卡住、最近是否在重复绕路等
- 宝箱特征：主要回答“现在哪个资源值得追”。 
  - 环境信息：宝箱是否还有效、和自己相对方向、相对距离
  - 先验指标：当前危险度大不大，宝箱是否处在值得去拿的位置
- 怪物特征：主要回答“危险从哪里来”。
  - **最近**怪物距离，**第二只怪物**的位置
  - 两只怪物方位带来的**局势**
    - > 因为很多时候真正的危险不是某一只怪贴脸，而是两只怪在压你的活动空间。
- 技能 / buff 特征：主要回答“我现在还有没有操作空间”
  - 环境信息：闪现是否可用、buff 有没有拿到
  - 先验指标：现在是该贪一点还是该保守一点
- 局部地图特征：主要回答“我往哪边走更安全”。 
  - 先验特征：附近有没有开阔区域、有没有长走廊、有没有死角

### P佬重视的几类信息

> P佬更在意的是 **决策相关特征**，而不是把环境原始字段尽量全塞进去。

- 最近怪物距离
- 第二只怪物距离
- 最近宝箱方向和距离
- 当前闪现是否可用
- 距离怪物加速还有多久
- 当前是否已经进入高压阶段
- 周围地形是不是容易卡死

## 奖励构造

> 与上面监控指标一样，前后期分开设置

- 很关键的改动：把前期和后期拆开看，认为这个赛题前后期目标并不完全一样。
- 核心思路：
  - 前期更鼓励 拿资源 + 稳定推进
  - 后期更鼓励 保命 + 拉空间
  - 闪现奖励的不是“用了”，而是“用得值”
- 本质上服务同一件事：让模型学会在“继续拿分”和“赶紧保命”之间切换。

### 主要奖励构成

- 显式部分
  - `+`🔊 生存：每步固定值，保证模型在没有拿到显式收益时也能收到“先活着”的信号
  - `+`🔊 步数分：环境步数得分 真正上涨时给
  - `+`🔉  宝箱：环境宝箱得分 真正上涨时给
  - `+`🔉  buff：环境buff收集数量 真正上涨时给
  - `±`🔊 怪物距离：到最近怪物的距离变远近而正负
- 隐式部分(逻辑)
  - `+`🔊 宝箱接近：到最近宝箱的距离缩短 + **环境不危险**
  - `?`🔊 临近加速缓冲：临近怪物加速阶段时，根据离最近怪物、第二只怪物距离，鼓励模型预留空间
  - `?` 🌡 后期生存系数：加速阶段，明显提高 “怪物距离” 和 “维持安全空间” 项的权重
  - `+`🔊 所处走廊安全性：根据当前位置周围几个方向的通路长度、平均可走深度以及附近区域是否开阔来算，位置越开阔、越容易拉扯，奖励越高
  - `-`🔊 包夹惩罚：根据两只怪物和自己的相对位置关系判断是否有合围趋势，活动空间是否被两只怪一起压缩
  - `-`🔊 死角惩罚：当前位置是否开阔、内圈是否容易转身，越像死角或短胡同，惩罚越重
  - `-`🔉  无效移动惩罚：某动作后未产生有效位移则惩罚(撞墙、闪现冷却时闪现) 
  - `-`🔊 重复探索惩罚：最近一段时间是否一直在老区域里绕圈，重复越久惩罚越明显
  - `+`🔊 闪现脱险奖励：使用闪现后和怪物距离显著拉开；自己到了更安全开阔的位置
  - `-`🔊 闪现滥用惩罚：使用闪现后未脱险、未改善占位、未获取资源
  - `-`🔊 第二只怪的压力惩罚：单独看第二只怪物的距离，过近则额外补一层惩罚，避免只盯着最近的
  - `+`🔉  回合终奖励：统一结算，区分正常完成、异常截断、阵亡等情况

- 认为奖励中最关键的部分
  - 怪物距离 shaping：这是最核心的生存信号之一，尤其影响后期质量。
  - 宝箱分奖励 / 宝箱接近奖励：这是前期拿资源的主要驱动力。
  - 临近加速缓冲奖励：这个很影响模型能不能顺利过渡到高压阶段。
  - corridor 奖励 + 死角惩罚：这两项对后期是否容易暴毙帮助很明显。
  - 闪现脱险奖励：能不能把闪现学成“保命技能”而不是乱交技能，这项很关键。

## 算法和架构

### 算法 PPO

- 没有对算法本身做改动
- 重要参数
  - `gamma = 0.995`
  - `lamda = 0.95`
  - `clip = 0.2`

### 网络架构

> 毕竟常规的做法

结构尽量贴任务本身

拆成几部分去处理：

- 英雄自身状态
- 宝箱信息
- 怪物信息
- buff / 技能信息
- 局部地图信息

P佬认为，这个题里“谁在威胁我”“哪个目标更值得追”“周围有没有逃生路线”都很重要，所以输入最好有一点结构，而不是完全拍扁。

网络主要思路：

- 标量特征交给 MLP
- 局部地图交给卷积
- 多个实体的信息先分别编码，再做融合

> 👇 这个注意力是如何实现的呢

另外我这里也稍微用了点注意力的思路，做法上更接近“先把宝箱、怪物、buff 这些实体各自编码成 embedding，再让模型自己决定当前更该看哪一类实体”。这样前期更容易把注意力放在资源上，后期怪物压力上来之后，也更容易把注意力切到怪物和逃生空间上。

## 训练流程

### train / val split

> P佬今年强烈建议的，也是我完全忽视的部分，属于是没亲自跑空想很容易丢

- Insight：
  - 很多改动看起来 train 在涨，但 **val 根本没涨**
  - 对训练地图的过拟合，导致评估地图上表现极差

P佬直接按地图拆的：

- 训练地图：`1~8`
- 验证地图：`9~10`

然后每隔 `10` 个 episode 跑 `1` 局 val。

### 课程学习

- P佬课程学习设置大致如下，目的比较直接：
  - 先让模型在资源更多、压力更低的环境里学会稳定推进
  - 再逐步把宝箱下限放低、buff 变少、第二只怪提前、怪物更早加速
  - 让模型从“会拿分”过渡到“高压下也能活”

## 总结

P佬觉得最有效的几个方向：

1. 把 observation 从“字段堆砌”改成“围绕决策重构”
2. 把 reward 按前后期目标拆开理解
3. 加入 train / val split，别只看 train 曲线
4. 用 curriculum 和随机化改善训练分布
5. 监控里一定要关注前后期差异

我自己的感受是，这题的关键不是某一个模块特别花，而是：让模型真正学会在“拿资源”和“保命”之间切换。
