# 2. reward 设计清单

## 一、分析

### 1. 这一部分在整个任务里的作用
`reward` 的作用是把“希望模型学会什么行为”明确传递给训练过程。对于这个任务来说，奖励不仅要覆盖显式得分，还要帮助模型建立生存、拿资源、使用闪现、处理高压阶段的行为偏好，促使模型学会在不同阶段切换策略。

### 2. 设计目标
- 在保留环境原始得分信号的基础上，补充与决策质量相关的 shaping。
- 区分前期推进与后期保命这两个不同阶段的行为目标。
- 强化真正有价值的操作，例如有效拉开怪物距离、合理吃箱子、合理使用闪现。
- 抑制低质量行为，例如无效移动、错误贪分、走入死角、闪现滥用。

### 3. 和其他模块的边界
- 与 `extractor` 的边界：`extractor` 提供奖励计算需要的状态和缓存，`reward` 定义如何把这些状态转成训练信号。
- 与 `obs` 的边界：`obs` 负责让模型“看懂局面”，`reward` 负责让模型“偏向正确行为”。
- 与 `algorithm_structure` 的边界：`reward` 不决定网络结构，但会影响训练稳定性和目标偏向。
- 与 `workflow` 的边界：`reward` 的有效性需要通过阶段性指标和 train/val 结果验证。



### 4. 结构性

#### 4.1 奖励设计总原则
- 以环境原始得分为主目标，不偏离比赛最终评分。
- shaping 的职责是提前暴露危险信号、补充推进信号、修正低质量动作，而不是替代任务目标。
- 奖励设计要服务于“拿资源”和“保命”之间的阶段性切换。
- 优先使用与决策强相关、解释性强、易稳定实现的奖励项。

#### 4.2 前后期分阶段奖励思路

- $$总奖励 = \alpha * 生存项奖励 + (1-\alpha)*探索资源项奖励 + 论外奖励项$$

- 统一阶段权重 $$α$$：随阶段推进权重递增
  - 单怪物
  - 2怪物出现
  - 怪物加速

- 前期更强调资源获取、推进、探索与建立有利位置。
- 后期更强调保命、拉空间、规避夹击与合理保留技能。

#### 4.x 标注修饰

- 部分奖励组件属于系数，用🌡修饰，有些属于值，用🔊🔉修饰
- 值奖励组件会使用 `↑` `↓` 标注其权重受 $$\alpha$$ 还是$$1-\alpha$$ 修饰
- 值奖励组件中，🔊表示稠密奖励(每步都有或某整个阶段有)，🔉表示稀疏奖励(仅特定条件下存在)
- 值奖励组件中，会使用 `+`、`-`、`± ` 来确定值的正负性(均包含0，即非负、非正)

#### 4.3 环境原始奖励

> 终局结果仍应与最终对局得分和完成情况保持一致

- `+↑`🔊 环境步数得分增量 /100
- `+↓`🔉 环境宝箱得分增量 /100

#### 4.4 生存相关 shaping

- 🌡 最近怪物距离：修饰距离变化
- `±↑`🔊 最近怪物距离变化奖励/惩罚
- 🌡 两只怪物平均距离：修饰夹击趋势
- `-↑`🔊 夹击趋势 (暂定两个怪物的方位夹角，验证轨迹预测后可改为寻路尾端方向夹角)
- `±↑`🔊 活动空间较上步的增减
- `-↑`🔉 进入死角、短走廊、低开阔区域的惩罚

#### 4.5 资源获取与推进 shaping
- `±↓`🔊 是否较上步更接近最近宝箱
- `+↓`🔉 获取宝箱的奖励，环境原始奖励给过了
- 获取 buff 的奖励：buff本身就会提高步数分增量，避免重复给，但其对后期生存确实重要，持不确定态度
- `+↓`🔊 未知区域的探索增量

#### 4.6 技能与动作质量 shaping
- `-↑`🔉 无效移动惩罚(移动未带来位移，包含了原地闪现)
- `-↑`🔉 低价值闪现：闪现位移小于0.5倍最长位移
- `+↑`🔉 闪现收益补充：脱险或进入更优位置。(
- `-↑`🔉 低效移动惩罚：根据自身中心3*3格子(随便给的一个)的访问次数判定，是否统计重复绕路、低效移动

#### 4.7 终局奖励与异常情况处理
- 异常截断特殊处理
- 步数上限：最优质的结局
- 阵亡：
  - 阶段阵亡：1、2、3
  - 夹击阵亡(2·3阶段)：
  - 死路阵亡(1-3阶段)：
  - 怪物同侧被追赶上(2-3阶段)：次优结局


#### 4.8 `extra_info` 的使用边界
- `extra_info` 可用于训练期的监控、诊断与部分辅助 shaping
- 可用于更稳定地评估危险度、空间风险和终局局势
- 不应直接把测试时不可感知的真实资源位置或全局最优答案转化为强监督奖励
- 若使用 `extra_info`，应确保奖励是在辅助学习局势，而不是替 observation 偷偷补答案

## 二、具体属性

### 1. 必须项

#### 1.1 环境原始奖励
| 名称 | 来源 |
|------|------|
| 环境步数得分增量 /100 | 环境原始 reward 或 score 增量 |
| 环境宝箱得分增量 /100 | 环境原始 reward 或 treasure score 增量 |

#### 1.2 分阶段权重与奖励组合
| 名称 | 来源 |
|------|------|
| 阶段权重 $$\alpha$$ | 当前步数 / 当前阶段 / 关键事件节点派生 |
| 生存项奖励总和 | 生存相关 shaping 聚合 |
| 探索资源项奖励总和 | 资源获取与推进 shaping 聚合 |
| 论外奖励项 | 技能质量 / 终局奖励 / 特殊修正项聚合 |
| 总奖励 | $$\alpha * 生存项奖励 + (1-\alpha) * 探索资源项奖励 + 论外奖励项$$ |

#### 1.3 生存相关 shaping
| 名称 | 来源 |
|------|------|
| 最近怪物距离修饰系数 | 当前帧最近怪物与英雄距离 |
| 最近怪物距离变化奖励/惩罚 | 当前帧与上一帧最近怪物距离差 |
| 两只怪物平均距离修饰系数 | 当前帧两只怪物与英雄距离的平均值 |
| 夹击趋势惩罚 | 两只怪物相对英雄方位关系 × 距离修饰派生 |
| 活动空间较上步的增减 | 当前帧与上一帧可行动空间比较 |
| 进入死角、短走廊、低开阔区域的惩罚 | 局部地图结构 / 空间先验派生 |

#### 1.4 资源获取与推进 shaping
| 名称 | 来源 |
|------|------|
| 是否较上步更接近最近宝箱 | 当前帧与上一帧最近已知宝箱距离比较 |
| 获取宝箱的奖励 | 宝箱收集事件 / treasure score 增量 |
| 获取 buff 的奖励（暂定观察项） | buff 获取事件 / buff 状态变化 |
| 未知区域的探索增量 | 当前帧与上一帧探索区域差异 |

#### 1.5 技能与动作质量 shaping
| 名称 | 来源 |
|------|------|
| 无效移动惩罚 | 动作后位置未变化 / 非法动作结果 |
| 低价值闪现惩罚 | 闪现位移长度与最长可达位移比较 |
| 闪现收益补充 | 闪现后怪物压力变化 / 空间改善 / 是否脱险 |
| 低效移动惩罚 | 局部访问次数 / 短窗口路径重复度 |

#### 1.6 终局奖励与异常情况处理
| 名称 | 来源 |
|------|------|
| 异常截断特殊处理 | `done` 原因 / 环境终止标记 / 训练框架回传信息 |
| 步数上限终局奖励 | 达到最大步数 / 正常完成事件 |
| 阶段阵亡终局惩罚 | 阵亡时刻对应阶段 |
| 夹击阵亡终局惩罚 | 阵亡时双怪相对方位与空间关系派生 |
| 死路阵亡终局惩罚 | 阵亡时局部地图结构与可逃方向派生 |
| 怪物同侧追赶上阵亡终局惩罚 | 阵亡前怪物追击关系与相对方向派生 |
