# 2. reward 设计清单

## 一、分析

### 1. 这一部分在整个任务里的作用
`reward` 的作用是把“希望模型学会什么行为”明确传递给训练过程。对于这个任务来说，奖励不仅要覆盖显式得分，还要帮助模型建立生存、拿资源、使用闪现、处理高压阶段的行为偏好，促使模型学会在不同阶段切换策略。

### 2. 设计目标
- 在保留环境原始得分信号的基础上，补充与决策质量相关的 shaping。
- 区分前期推进与后期保命这两个不同阶段的行为目标。
- 强化真正有价值的操作，例如有效拉开怪物距离、合理吃箱子、合理使用闪现。
- 抑制低质量行为，例如无效移动、错误贪分、走入死角、闪现滥用。

### 3. 和其他模块的边界
- 与 `extractor` 的边界：`extractor` 提供奖励计算需要的状态和缓存，`reward` 定义如何把这些状态转成训练信号。
- 与 `obs` 的边界：`obs` 负责让模型“看懂局面”，`reward` 负责让模型“偏向正确行为”。
- 与 `algorithm_structure` 的边界：`reward` 不决定网络结构，但会影响训练稳定性和目标偏向。
- 与 `workflow` 的边界：`reward` 的有效性需要通过阶段性指标和 train/val 结果验证。
