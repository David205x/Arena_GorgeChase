# 3. algorithm_structure 设计清单

## 一、分析

### 1. 这一部分在整个任务里的作用
`algorithm_structure` 负责把已经定义好的观测和奖励，映射为一个适合训练和推理的算法与网络实现框架。它的重点不是发明复杂算法，而是确保输入组织、编码方式和策略/价值学习过程与任务结构相匹配。

### 2. 设计目标
- 采用稳定可靠的 PPO 作为训练主算法。
- 让网络输入结构尽量贴近任务本身，而不是把所有信息完全无结构拍扁。
- 区分不同类型输入的编码方式，例如标量、实体、局部地图。
- 为后续扩展留出空间，例如增加更强的实体融合或阶段感知模块。

### 3. 和其他模块的边界
- 与 `obs` 的边界：`obs` 决定输入内容，`algorithm_structure` 决定如何编码和融合这些输入。
- 与 `reward` 的边界：`reward` 决定训练目标，`algorithm_structure` 决定模型如何去拟合这个目标。
- 与 `extractor` 的边界：`extractor` 提供数据层支持，`algorithm_structure` 不直接关心底层原始字段。
- 与 `workflow` 的边界：`algorithm_structure` 不负责训练调度与评估流程，但其好坏要通过 workflow 中的指标与实验结果体现。
