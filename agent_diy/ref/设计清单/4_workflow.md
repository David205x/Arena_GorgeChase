# 4. workflow 设计清单

## 一、分析

### 1. 这一部分在整个任务里的作用
`workflow` 负责定义整个训练与评估过程如何组织，确保我们不是只看到训练曲线变好，而是真正能够判断模型是否学会了前后期切换、是否能适应高压阶段、是否具备泛化能力。它是把 observation、reward、algorithm 落到实验闭环中的部分。

### 2. 设计目标
- 建立能反映真实学习效果的训练、验证、监控与对比流程。
- 区分训练提升、泛化提升和阶段性能力提升。
- 通过 train/val split、课程学习和分阶段指标，避免错误判断模型质量。
- 为后续调参和消融实验提供统一流程模板。

### 3. 和其他模块的边界
- 与 `obs` 的边界：`workflow` 不定义观测内容，但负责验证观测设计是否有效。
- 与 `reward` 的边界：`workflow` 不定义奖励本身，但负责通过指标判断奖励是否把模型引向了正确方向。
- 与 `algorithm_structure` 的边界：`workflow` 不决定网络结构，但负责比较不同结构和超参配置的效果。
- 与 `extractor` 的边界：`workflow` 不直接处理底层数据实现，只消费各模块输出并形成训练闭环。
